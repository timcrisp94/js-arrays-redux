Array Iterator Methods are higher order functions

They are higher order functions because they take functions as arguments

  • Higher order functions delegate task
  • Higher order functions are composable
    • shorter syntax
    • less prone to errors
    • declarative

    Hot Iterator Methods

    • .map() transforms the array and returns a new transformed array

    • .reduce() "the multitool of all higer order functions" - Shahzad

    • .filter() creates a new array for us, returns the filtered items only. the callback fn returns a boolean value to the filter method

    By far the hottest

    Shah's solution, given an object myTransactions, for figuring out the total amount of money spent on dining

    const totalSpendingOnDining = myTransactions

    .filter(transaction => transaction.type === 'charge')

    .filter(transaction => transaction.category === 'dining')

    .map(transaction => transaction.amount)

    .reduce((prev, amount) => (prev || 0) + amount

    console.log(`The total charge for dining is $ ${totalSpendingOnDining}`)
